# PRD: 用药手册配色重新设计

## Introduction

将用药手册从当前暗黑主题重新设计为适合医疗健康应用的浅色主题。目标是提升用户体验，营造温暖、信任、专业的感觉。

## Goals

- 从暗黑主题改为浅色主题（浅灰/白底）
- 使用医疗蓝#4A90E2和健康绿#7ED3B2作为主色调
- 增加卡片层次感（阴影、边框）
- Icon设计体现功能特点
- 保持所有功能不变，仅改视觉

## User Stories

### US-001: 设计配色系统CSS变量
**Description:** 作为开发者，我需要定义统一的配色CSS变量，便于全局应用和后续调整。

**Acceptance Criteria:**
- [ ] 定义主色调：医疗蓝#4A90E2、健康绿#7ED3B2
- [ ] 定义背景色：浅灰#F5F7FA、白色#FFFFFF
- [ ] 定义文字色：主文字#333333、次要文字#666666、辅助文字#999999
- [ ] 定义阴影色：轻阴影rgba(0,0,0,0.05)、重阴影rgba(0,0,0,0.1)
- [ ] 定义成功/警告/错误色：#7ED3B2/#FFB347/#FF6B6B
- [ ] 在所有页面顶部统一使用这些CSS变量
- [ ] 创建colors.css文件，统一管理配色变量
- [ ] 所有页面引用colors.css
- [ ] 在浏览器中验证：变量定义正确，无语法错误

## Functional Requirements

- FR-1: 创建colors.css统一管理配色变量
- FR-2: 所有页面引用colors.css
- FR-3: 背景色统一为浅灰#F5F7FA或白色#FFFFFF
- FR-4: 主色调为医疗蓝#4A90E2和健康绿#7ED3B2
- FR-5: 文字色为#333/#666/#999三级
- FR-6: 卡片使用白色背景+轻阴影+圆角8px
- FR-7: Icon使用医疗蓝或健康绿，保持统一风格
- FR-8: 保持所有功能不变（搜索、查看、提醒、问答）
- FR-9: 增加卡片层次感（阴影、边框）

## Non-Goals

- 不改变页面结构和布局
- 不添加新功能
- 不改变交互逻辑
- 不优化性能（仅配色调整）

## Technical Considerations

- 使用CSS变量便于全局调整
- 保持响应式设计不变（max-width: 375px）
- Icon使用FontAwesome或内联SVG
- 阴影使用rgba颜色，便于调整透明度
- 保持所有JavaScript功能不变

## Success Metrics

- 用户反馈配色舒适、温暖、专业
- 所有页面功能保持不变
- 代码统一使用CSS变量，便于后续调整
- 所有页面在浏览器中正常打开

## Open Questions

- 是否需要深色模式切换？（当前不实现，未来可考虑）
- Icon是否使用FontAwesome还是内联SVG？（推荐内联SVG，减少依赖）
- 是否需要配色预览页面？（可考虑添加，便于整体查看）
